🐢 Графика Черепахи: Прямоугольник в Кумир

Программа на языке Кумир (Kumir) для рисования прямоугольника с использованием исполнителя «Черепаха»:

использовать Черепаха

алг Прямоугольник

нач
    опустить хвост

    нц 2 раз
        вперед(200)
        вправо(90)
        вперед(100)
        вправо(90)
    кц

кон

📝 Как работает программа:

  • использовать Черепаха — подключение модуля графического исполнителя
  • алг Прямоугольник — объявление алгоритма с именем «Прямоугольник»
  • опустить хвост — включение режима рисования: при движении черепаха оставляет след
  • нц 2 раз ... кц — цикл повторения 2 раза (достаточно для замкнутого прямоугольника)
  • вперед(200) — движение вперёд на 200 шагов (длинная сторона прямоугольника)
  • вправо(90) — поворот направо на 90 градусов
  • вперед(100) — движение вперёд на 100 шагов (короткая сторона прямоугольника)
  • вправо(90) — повторный поворот для продолжения контура
  • кон — завершение алгоритма

🔧 Возможные модификации программы
  • Изменить размеры: замените 200 и 100 на нужные значения
  • Нарисовать квадрат: установите одинаковые значения: вперед(150) дважды
  • Добавить цвет: перед опустить хвост добавьте цвет(красный)
  • Сместить начало: добавьте сместиться(х, у) в начале алгоритма
  • Нарисовать несколько фигур: оберните весь код в дополнительный цикл нц N раз
  • Изменить угол: для ромба используйте вправо(60) и вправо(120)
📚 Синтаксис Кумир — кратко:
• Ключевые слова: алг, нач, кон, нц, кц — пишутся строчными буквами
• Комментарии: начинаются с | или заключаются в { }
• Отступы: не обязательны, но рекомендуются для читаемости
• Имена алгоритмов: пишутся слитно, с заглавной буквы, без пробелов

🗄️ База данных: Иерархическая модель медицинской системы

Разработка иерархической модели данных для учета пациентов, хирургов и операций:

📋 Описание модели:

  • ПАЦИЕНТ: номер, имя, адрес, группа крови
  • ХИРУРГ: номер патента, имя
  • ОПЕРАЦИЯ: дата, тип операции, препарат, побочный эффект
🔗 Связи в базе данных:

1. ПАЦИЕНТ → ОПЕРАЦИЯ (один-ко-многим)
Один пациент может перенести несколько операций. Каждая операция связана с одним конкретным пациентом через номер пациента (внешний ключ).

2. ХИРУРГ → ОПЕРАЦИЯ (один-ко-многим)
Один хирург может проводить множество операций. Каждая операция выполняется одним хирургом, который указывается через номер патента хирурга (внешний ключ).

3. ПАЦИЕНТ → ХИРУРГ (многие-ко-одному)
Пациент наблюдается у конкретного хирурга. Связь показывает, какой хирург ведет пациента.

Тип связей: Иерархические (древовидные) с родительскими сущностями (ПАЦИЕНТ, ХИРУРГ) и дочерней сущностью (ОПЕРАЦИЯ).

📊 Диаграмма: Алгоритм обучения нейронной сети

Блок-схема алгоритма обучения персептрона с учителем:

📝 Описание алгоритма:

  • Инициализация: случайная генерация весов W₀
  • Входные данные: обучающие примеры x(k), d(k), где k = 1,p
  • Параметры: η - скорость обучения, N - число входов, M - число выходов
  • Цикл обучения: для каждого примера вычисляется выход y(k) и ошибка e(k)
  • Коррекция весов: wᵢ := wᵢ - η·e(k)·x(k)
  • Критерий остановки: S_err ≤ E_son (достигнута требуемая точность)
💡 Примечание: Данный алгоритм реализует метод обучения персептрона с использованием правила дельта (delta rule) или метода наименьших квадратов. Алгоритм итеративно корректирует веса сети до достижения заданной точности E_son.
